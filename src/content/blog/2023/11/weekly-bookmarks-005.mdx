— I was a guest on a podcast, where I chatted with [Brian Kardell](https://bkardell.com/) and [Eric Meyer](https://meyerweb.com/) about some CSS stuff. You can listen to it if you’d like. Or read it — the transcript is available as well.

## Web Components and Shadow DOM

In the last few weeks, there were _many_ articles about web components, custom elements and shadow & light DOMs. I won’t link to _all of them_, as that would be too much, but to a snippet of them, in chronological order. Most of these articles would have links to other related recent posts — highly recommend!

- [“Web Components Will Outlive Your JavaScript Framework”](https://jakelazaroff.com/words/web-components-will-outlive-your-javascript-framework/) by [Jake Lazaroff](https://jakelazaroff.com/) — an article about how using native web components can make the code more resilient (given we won’t use any framework code inside of them).
- [“Blinded By the Light DOM”](https://meyerweb.com/eric/thoughts/2023/11/01/blinded-by-the-light-dom/) by [Eric Meyer](https://meyerweb.com/) — an article about how we don’t have to use Shadow DOM to use Web Components, and can just rely on light DOM, _progressively enhancing_ our regular HTML. The next day after this article, I published what can be called an inverse article: [“Shadow DOM on Regular Elements”](/shadows-separately/) — how we can do the opposite and not use custom elements, but still use shadow DOM.
- [“Presentational shadow DOM”](https://www.mayank.co/blog/presentational-shadow-dom/) by [Mayank](https://www.mayank.co/) — an article about how shadow DOM makes it harder to use the cascade, where they propose some hypothetical ways to improve this.
- [“HTML web components”](https://adactio.com/journal/20618) by [Jeremy Keith](https://adactio.com/) — an article that coins the term, and links to numerous other articles (a lot of them are not in my list).
- [“HTML Web Components are Just JavaScript?”](https://www.oddbird.net/2023/11/17/components/) by [Miriam Suzanne](https://www.miriamsuzanne.com/) — an article exploring some aspects of web components, as well as a very curious idea of a `register-element` web component to register other web components. I really hope at one point we will get the [Declarative Custom Elements](https://github.com/WICG/webcomponents/issues/1009), but I really like an idea to use another web component to do this today!
