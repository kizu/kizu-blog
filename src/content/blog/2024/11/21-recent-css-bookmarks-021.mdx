Anchor positioning is cool! And there were many articles and posts about anchor positioning since I last mentioned it in my bookmark posts. Here they are!

<aside class="warning">
While both Apple and Mozilla are working on anchor positioning right now, most of the examples from these articles will work only in Chrome (and/or Chrome Canary) for now.
</aside>

- [“Let’s hang! An intro to CSS Anchor Positioning with basic examples”](https://utilitybend.com/blog/lets-hang-an-intro-to-css-anchor-positioning-with-basic-examples) by [Brecht De Ruyte](https://utilitybend.com/) — a good introductory article about anchor positioning that compares two different ways we can achieve the same thing, and looks into some other details.

- [“Anchoreum”](https://anchoreum.com/) by [Thomas Park](https://thomaspark.co/) — a fun game that teaches anchor positioning. I did not yet finish it, but it might be a good way to learn how some of its aspects work if you’re not yet familiar with it.

- [“Layered positioning”](https://github.com/w3c/csswg-drafts/issues/9816) by [Christoph Päper](https://github.com/Crissov) — a CSSWG issue about how we could make an anchored element match the layer of its anchor. I [commented there today](https://github.com/w3c/csswg-drafts/issues/9816#issuecomment-2492457219) with two small proposals, the ideas for which I got after a [mastodon thread](https://front-end.social/@mayank/113376766689246458) started by [Mayank](https://www.mayank.co/). Check out the thread itself for a demo of another anchor positioning use case: animated focus ring. I mentioned this use case in [my CSSDay talk](https://www.youtube.com/watch?v=ZC_WS05t2lI), but did not have an opportunity to finish a draft of an article about it yet.

- [“Anchor Positioning Is Disruptive”](https://www.oddbird.net/2024/11/18/anchor-position-yearbook/) by [James Stuckey Weber](https://jamessw.com/) — a post about an interesting use case for anchor positioning, a bit similar to the one with the sidenotes. See also [“Another Anchor Positioning Demo: Multiple Anchors”](https://css-irl.info/another-anchor-positioning-demo-multiple-anchors/) by [Michelle Barker](https://css-irl.info/), [Nuclear Anchored Sidenotes](https://meyerweb.com/eric/thoughts/2023/09/12/nuclear-anchored-sidenotes/) by [Eric Meyer](https://meyerweb.com/), and the [“Sidenotes Layout”](https://kizu.dev/anchor-positioning-experiments/#sidenotes-layout) section of my old [“Future CSS: Anchor Positioning”](https://kizu.dev/anchor-positioning-experiments/) article.

- [“Custom Range Slider Using Anchor Positioning & Scroll-Driven Animations”](https://frontendmasters.com/blog/custom-range-slider-using-anchor-positioning-scroll-driven-animations/) by [Temani Afif](https://css-articles.com/) — an article about how we can target pseudo-elements of the range slider with both scroll-driven animations and anchor positioning. The result is very impressive and even uses the technique from the [“Solved by CSS Scroll-Driven Animations: Style an element based on the active Scroll Direction and Scroll Speed”](https://www.bram.us/2023/10/23/css-scroll-detection/) article by [Bramus](https://www.bram.us/). I am glad that we now have a proper exploration of this concept, which I shared a year ago in my [“Observation: Range Input’s Thumb and Emerging CSS”](https://blog.kizu.dev/input-range-thumb/) post.

- [“Custom Progress Element Using Anchor Positioning & Scroll-Driven Animations”](https://frontendmasters.com/blog/custom-progress-element-using-anchor-positioning-scroll-driven-animations/) by [Temani Afif](https://css-articles.com/) — a continuation of the previous article, this time applying this method to the `<progress>` HTML element.

- [“Front-End solution: Eyebrow heading dots”](https://piccalil.li/blog/front-end-solution-eyebrow-heading-dots/) by [Andy Bell](https://andy-bell.co.uk/) — a post about a challenge for placing elements on two sides of a wrapping element with a solution that builds on top of my [“The Shrinkwrap Problem: Possible Future Solutions”](https://kizu.dev/shrinkwrap-problem/) article that I shared with the author.

- [“Anchor Positioning and the Popover API for a JS-Free Site Menu ”](https://css-irl.info/anchor-positioning-and-the-popover-api/) by [Michelle Barker](https://css-irl.info/) — an article that looks at various aspects of how we could use anchor positioning with popover API, including a site menu use case.

- [“Progressively Enhanced Popover Toggletips”](https://css-irl.info/progressively-enhanced-popover-toggletips/) by [Michelle Barker](https://css-irl.info/) — an article about using anchor positioning to progressivly enhance footnotes.

- [“Web components + anchor positioning + popover API = I love the platform”](https://jschof.dev/posts/2024/9/love-letter-to-the-platform/) by [Jim Schofield](https://jschof.dev/) — a post that looks at how we could build a popover with anchor positioning as a custom element.

- [“Flow Charts with CSS Anchor Positioning”](https://coryrylan.com/blog/flow-charts-with-css-anchor-positioning) by [Cory Rylan](https://coryrylan.com/) — a post that looks at the flow charts use case for anchor positioning. See also: the [“Using the Connections”](https://kizu.dev/anchor-positioning-experiments/#using-the-connections) section of my old [“Future CSS: Anchor Positioning”](https://kizu.dev/anchor-positioning-experiments/) article.
