As usual, a few articles about CSS that I found interesting this week, nine of them.

- [“Proposal: CSS Variable Groups”](https://lea.verou.me/docs/var-groups/) by [Lea Verou](https://lea.verou.me/) — a proposal and [an associated CSSWG issue](https://github.com/w3c/csswg-drafts/issues/9992) about an ability to have something similar to maps or arrays in native CSS. I wanted to have something like this for years, and I truly hope that this will lead to something!

- [“Okay, Color Spaces”](https://ericportis.com/posts/2024/okay-color-spaces/) by [Eric Portis](https://ericportis.com/) — a very comprehensive article about the color spaces which makes it easy to understand how they work and what are the differences between them.

- [“CSS `:has()` Interactive Guide”](https://ishadeed.com/article/css-has-guide/) by [Ahmad Shadeed](https://ishadeed.com/) — a detailed article with many interactive examples about what `:has()` pseudo-class is, and many of its use cases.

- [“Using container size and style queries”](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_containment/Container_size_and_style_queries) by [Estelle Weyl](https://standardista.com/) — an MDN guide about container queries (both size and style ones).

- [“View transitions: Handling aspect ratio changes”](https://jakearchibald.com/2024/view-transitions-handling-aspect-ratio-changes/) by [Jake Archibald](https://jakearchibald.com/) — a post explaning how we can improve or fix view transitions in complex cases by separating the transitions to happen on multiple elements.

- [“Speedier tunes”](https://adactio.com/journal/20899) by [Jeremy Keith](https://adactio.com/) — a short post about how `content-visibility` can improve performance when rendering lists with complex elements (in this case, with SVGs inside).

- [“Changes to CSS `::backdrop` inheritance”](https://developer.chrome.com/blog/css-backdrop-inheritance?hl=en) by [Bramus](https://www.bram.us/) — a post about the recent update to the `::backdrop`, allowing it to inherit styles from the originating element.

- [“A practical guide to using shadow DOM”](https://www.mayank.co/blog/declarative-shadow-dom-guide) by [Mayank](https://www.mayank.co/) — a post explaining what shadow DOM is, with a focus on the _declarative_ shadow DOM which is now available in all major browsers.

- [“Declarative Custom Elements Should Take Inspiration From CSS”](https://knowler.dev/blog/declarative-custom-elements-should-take-inspiration-from-css) by [Nathan Knowler](https://knowler.dev/) — a post outlining an idea to use a CSS-like syntax for re-binding HTML attributes declaratively. It did remind me of [Cascading Attribute Sheets proposal](https://web.archive.org/web/20220606213657/https://www.xanthir.com/blog/b4K_0) by [Tab Atkins-Bittner](https://xanthir.com/), though I doubt we will get anything like that anytime soon.
